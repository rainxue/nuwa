# 背景


# 数据类型
db_type：
- MySQL数据库：mysql
- 达梦数据库：dm

# 文件名规范
- 分模块存放数据库的ddl文件
- ddl文件存放路径：/src/nservers/{nserver_name}/ddl
- nserver的ddl文件命名规范-全量：ddl_{db_type}_all.sql
- nserver的ddl文件命名规范-增量：ddl_{db_type}_{yyyyMMdd}.sql、{db_type}_{yyyyMMdd}_2.sql

# 数据库设计规范
- 不要建任何外键约束
- id不要使用主键自增
- 同一个模块使用同一个表名前缀，前缀为模块名或模块名的缩写，不超过3个字符，再加下划线"_"，比如：模块sys，则数据表名前缀：sys_
- 字段名需要规避数据库的关键字和保留字，当前虽然只支持MySQL，但后续可能会支持更多的数据库，包括达梦、PostgreSQL、人大金仓等主流数据库或国内信创数据库
- 字段名和Schema中属性名一致，除非和数据库的关键字和保留字冲突，若未指定Schema，则注意字段名使用小写字母，多个单词使用下划线"_"分隔，比如：create_by、update_date
- 考虑到不是所有数据库都支持枚举类型，Schema中enum类型的字段，数据库中使用整形
- 考虑到不是所有数据库都支持JSON，Schema中json类型的字段，数据库中使用text类型
